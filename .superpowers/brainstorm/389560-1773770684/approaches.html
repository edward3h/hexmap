Backend Architecture Approaches

Three ways to add auth + write API to the existing PHP + Vite stack

A

Extend the PHP backend (Recommended)

Add OAuth2 (Discord + Google), sessions, and write endpoints directly to the existing PHP backend. New admin UI pages added to the Vite app at /admin.

Pros

  • Single backend to deploy and maintain
  • Consistent with existing shared-host deployment
  • No new runtime dependencies

Cons

  • PHP OAuth libraries are more verbose than JS equivalents
  • api.php grows larger (mitigated by splitting into files)
B

Add a Node.js/Express auth layer

Keep PHP for read data. Add a Node.js server handling OAuth2 and write operations. Admin UI in Vite calls both servers.

Pros

  • Mature OAuth libraries (Passport.js)
  • TypeScript consistency with frontend

Cons

  • Two backends to manage and deploy
  • Complicates shared-host deployment
  • Split responsibilities between PHP and Node
C

Migrate backend to Node.js/TypeScript entirely

Rewrite backend in Express + TypeScript. Handles auth, all read and write APIs. Deploy as a single Node process.

Pros

  • Full TypeScript consistency across the stack
  • Best OAuth ecosystem
  • Easier to evolve long-term

Cons

  • Significant migration effort (rewrite existing PHP endpoints)
  • May not fit shared-host PHP deployment
